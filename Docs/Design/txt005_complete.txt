2. Customization Model: Modular Parts

Player NetPs are built from swappable modular parts, not sliders/morphs. This maps directly onto Unreal's Customizable Object (Mutable) plugin, already present in the project.

2.2 Part Unlocking by Tier/Rank

Recommended pattern (pending confirmation): each part has a MinimumTier and/or MinimumRank requirement, gating it from the character creator's selectable list until the player's progression meets it. This reuses your existing Tier/Rank stat sheet categories as the unlock currency, rather than inventing a separate unlock system.

2.3 Part Coloring by Element

Each equipped part's material should expose an Element-driven color parameter, consistent with the panel/HUD color language already established (ABattleGridManager::TierColor, the per-side glow system). This means a single mesh (e.g., one gauntlet model) can visually read as Fire, Aqua, Plasma, etc. via material parameter swap rather than needing 10 separate gauntlet meshes — a major asset-production time-saver.
