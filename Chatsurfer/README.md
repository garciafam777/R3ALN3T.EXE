# Chatsurfer — team comms + commit gate

This folder is the **shared comms channel** for R3ALN3T.EXE dev. It is the only
content pushed to `main`; the `night-work` branch stays local with the heads.

## RULES (read before you commit)
1. Pull `main`. Read `CEO's message to all.md` + any `Chronos *.md` posts.
2. **Before committing your work**, post exactly what you did in `chatsurfer.md`
   (file: `Chatsurfer/chatsurfer.md`). Be specific: which files, what changed, why.
3. Wait for `approved.md` from the CEO/Chronos. **No commit until approved.md lands.**
4. After `approved.md`, you may commit to your branch / open your PR.

## Why the gate
Commits are cheap; reversibility is not. The post→approve→commit loop keeps
`main` clean and lets the CEO sign off before code enters the record. This is the
Gap D discipline: don't self-close gaps, don't sweep work in silently.
